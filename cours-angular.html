1 Introduction

Angular est un framework front-end développé par Google, écrit en TypeScript. Il fournit une structure complète pour construire des Single Page Applications (SPA) : routing, formulaires, HTTP, animations, etc.

Architecture générale

Une application Angular s'organise en couches :

  • Modules (NgModule) — regroupent et organisent le code
  • Composants — blocs d'UI (HTML + logique)
  • Services — logique métier, accès aux données
  • Directives / Pipes — transformations dans les templates
  • Router — navigation entre vues

CLI Angular

La CLI Angular accélère le développement :

# Créer un projet
ng new mon-projet

# Générer un composant
ng generate component pages/home
ng g c pages/home  # raccourci

# Générer un service
ng g service services/auth

# Générer un module
ng g module features/user --routing

# Lancer le serveur de développement
ng serve

# Build de production
ng build --configuration production

Structure d'un projet Angular

src/
├── app/
│   ├── app.module.ts        # module racine
│   ├── app.component.ts     # composant racine
│   ├── app-routing.module.ts
│   ├── components/
│   ├── pages/
│   ├── services/
│   └── models/
├── environments/
│   ├── environment.ts       # dev
│   └── environment.prod.ts  # production
└── main.ts                  # point d'entrée

2 NgModule

Le décorateur @NgModule est le cœur de l'organisation d'une application Angular. Il déclare quels composants/services appartiennent à quel bloc fonctionnel.

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppRoutingModule } from './app-routing.module';
import { AppComponent } from './app.component';
import { UserService } from './services/user.service';

@NgModule({
  declarations: [
    AppComponent,        // composants, directives, pipes de CE module
    HeaderComponent,
    MonPipe,
  ],
  imports: [
    BrowserModule,       // modules Angular ou tiers nécessaires
    AppRoutingModule,
    ReactiveFormsModule,
  ],
  exports: [
    HeaderComponent,     // expose HeaderComponent aux modules qui importent celui-ci
  ],
  providers: [
    UserService,         // services disponibles dans ce module
  ],
  bootstrap: [AppComponent], // composant racine (app.module.ts uniquement)
})
export class AppModule {}
PropriétéContenuObligatoire
declarationsComposants, directives, pipes appartenant à ce moduleNon
importsAutres modules dont ce module a besoinNon
exportsCe que ce module rend disponible aux modules qui l'importentNon
providersServices disponibles dans ce moduleNon
bootstrapComposant lancé au démarrage (racine seulement)Module racine
Règle clé Un composant ne peut être déclaré que dans un seul module. Pour le partager, on l'exporte depuis un module "shared" que les autres importent.

Module de fonctionnalité (Feature Module)

On découpe l'app en modules par fonctionnalité pour organiser le code et activer le lazy loading :

// user.module.ts
@NgModule({
  declarations: [UserListComponent, UserDetailComponent],
  imports: [CommonModule, UserRoutingModule],
})
export class UserModule {}

Shared Module

Un module partagé contient les composants/directives/pipes utilisés dans toute l'app :

@NgModule({
  declarations: [LoaderComponent, MonPipe],
  imports: [CommonModule],
  exports: [LoaderComponent, MonPipe, CommonModule], // tout ce qu'on veut partager
})
export class SharedModule {}

3 Composants

Un composant = un bloc d'UI autonome. Il associe une logique TypeScript, un template HTML et un style CSS.

import { Component, OnInit, ChangeDetectionStrategy } from '@angular/core';

@Component({
  selector: 'app-profil',       // <app-profil> dans le HTML parent
  templateUrl: './profil.component.html',  // fichier HTML externe
  styleUrls: ['./profil.component.scss'],   // CSS scopé à ce composant
  standalone: false,              // true = pas besoin de NgModule
  changeDetection: ChangeDetectionStrategy.OnPush, // optimisation
})
export class ProfilComponent implements OnInit {
  nom = 'Alice';

  ngOnInit() {
    // initialisation
  }
}
PropriétéDescription
selectorNom du tag HTML pour utiliser le composant. Peut être un attribut ([app-profil]) ou une classe (.app-profil)
templateUrlChemin vers le fichier HTML
templateHTML inline (à la place de templateUrl)
styleUrlsTableau de fichiers CSS — le style est encapsulé (n'affecte pas les autres composants)
stylesCSS inline (tableau de chaînes)
standaloneSi true, le composant se déclare lui-même sans module
changeDetectionDefault ou OnPush — stratégie de mise à jour de la vue

Composant Standalone Angular 14+

Permet de se passer de NgModule. Le composant déclare ses propres dépendances dans imports :

@Component({
  selector: 'app-carte',
  standalone: true,
  imports: [CommonModule, RouterModule], // imports directs
  templateUrl: './carte.component.html',
})
export class CarteComponent {}

4 Templates & Data Binding

Le data binding est le mécanisme qui synchronise le composant (TypeScript) et la vue (HTML).

Interpolation {{ }}

Affiche la valeur d'une propriété dans le HTML :

<!-- composant : nom = 'Alice' -->
<p>Bonjour {{ nom }} !</p>
<p>{{ 2 + 2 }}</p>
<p>{{ nom.toUpperCase() }}</p>

Property Binding [property]

Lie une propriété HTML à une valeur du composant (sens : composant → DOM) :

<!-- src vient du composant -->
<img [src]="imageUrl" [alt]="imageAlt" />
<button [disabled]="isLoading">Envoyer</button>
<input [value]="nom" />

Event Binding (event)

Écoute un événement DOM et appelle une méthode du composant (sens : DOM → composant) :

<button (click)="sauvegarder()">Sauvegarder</button>
<input (input)="onInput($event)" />
<form (ngSubmit)="onSubmit()"></form>

Two-Way Binding [(ngModel)]

Synchronisation bidirectionnelle entre l'input et la propriété. Nécessite FormsModule :

<!-- composant : nom = '' -->
<input [(ngModel)]="nom" />
<p>{{ nom }}</p>  <!-- se met à jour en temps réel -->
[(ngModel)] = "Banana in a box" C'est la combinaison de [ngModel]="nom" (property binding) + (ngModelChange)="nom=$event" (event binding). Le [()] est du sucre syntaxique.

Récapitulatif

SyntaxeSensUsage
{{ valeur }}Composant → VueAfficher du texte
[propriété]Composant → DOMAttributs dynamiques
(événement)DOM → ComposantRéagir aux actions utilisateur
[(ngModel)]Les deuxInputs de formulaire

5 Directives

Les directives modifient l'apparence ou le comportement des éléments DOM. Il en existe deux types : structurelles (modifient le DOM) et d'attribut (modifient l'apparence).

Directives structurelles

*ngIf

<div *ngIf="isLoaded">Contenu chargé</div>
<div *ngIf="user; else loading">{{ user.nom }}</div>
<ng-template #loading><p>Chargement...</p></ng-template>

*ngFor

<ul>
  <li *ngFor="let item of items; let i = index; trackBy: trackById">
    {{ i }} - {{ item.nom }}
  </li>
</ul>

trackBy est une optimisation importante : Angular ne recrée pas les éléments si l'ID n'a pas changé.

trackById(index: number, item: any) {
  return item.id;
}

*ngSwitch

<div [ngSwitch]="statut">
  <p *ngSwitchCase="'actif'">Actif</p>
  <p *ngSwitchCase="'inactif'">Inactif</p>
  <p *ngSwitchDefault>Inconnu</p>
</div>

Nouvelle syntaxe de flux de contrôle Angular 17+

Angular 17 introduit une nouvelle syntaxe plus lisible, directement dans les templates :

<!-- @if -->
@if (isLoaded) {
  <p>Chargé</p>
} @else {
  <p>Chargement...</p>
}

<!-- @for -->
@for (item of items; track item.id) {
  <li>{{ item.nom }}</li>
} @empty {
  <p>Aucun élément</p>
}

<!-- @switch -->
@switch (statut) {
  @case ('actif') { <span>Actif</span> }
  @default { <span>Inconnu</span> }
}

Directives d'attribut

ngClass

<!-- Objet : la clé est la classe, la valeur est la condition -->
<div [ngClass]="{ 'actif': isActif, 'erreur': hasError }"></div>

<!-- Tableau -->
<div [ngClass]="['classe-a', 'classe-b']"></div>

<!-- Raccourci -->
<div [class.actif]="isActif"></div>

ngStyle

<div [ngStyle]="{ 'color': couleur, 'font-size.px': taille }"></div>

<!-- Raccourci -->
<div [style.color]="couleur"></div>

Créer une directive personnalisée

import { Directive, ElementRef, HostListener, Input } from '@angular/core';

@Directive({
  selector: '[appSurbrillance]' // s'utilise comme attribut
})
export class SurbrillanceDirective {
  @Input() appSurbrillance = 'yellow';

  constructor(private el: ElementRef) {}

  @HostListener('mouseenter')
  onEnter() {
    this.el.nativeElement.style.backgroundColor = this.appSurbrillance;
  }

  @HostListener('mouseleave')
  onLeave() {
    this.el.nativeElement.style.backgroundColor = '';
  }
}

<!-- Utilisation -->
<p [appSurbrillance]="'lightblue'">Survolez-moi</p>

6 Pipes

Les pipes transforment les données dans les templates, sans modifier la source. Syntaxe : valeur | nomPipe:argument.

Pipes intégrés

<!-- Texte -->
<p>{{ 'angular' | uppercase }}</p>        <!-- ANGULAR -->
<p>{{ 'ANGULAR' | lowercase }}</p>        <!-- angular -->
<p>{{ 'hello world' | titlecase }}</p>    <!-- Hello World -->

<!-- Nombre -->
<p>{{ 3.14159 | number:'1.2-2' }}</p>    <!-- 3.14 -->
<p>{{ 1500 | currency:'EUR':'symbol' }}</p> <!-- €1,500.00 -->
<p>{{ 0.75 | percent }}</p>              <!-- 75% -->

<!-- Date -->
<p>{{ today | date:'dd/MM/yyyy' }}</p>
<p>{{ today | date:'fullDate':'':'fr' }}</p>

<!-- Objet -->
<p>{{ obj | json }}</p>                   <!-- debug -->

<!-- Observable -->
<p>{{ user$ | async }}</p>               <!-- souscrit et unsubscribe automatiquement -->

<!-- Tranche -->
<p>{{ items | slice:0:3 | json }}</p>    <!-- 3 premiers éléments -->

Pipe personnalisé

import { Pipe, PipeTransform } from '@angular/core';

@Pipe({ name: 'truncate' })
export class TruncatePipe implements PipeTransform {
  transform(value: string, limit: number = 50): string {
    if (!value) return '';
    return value.length > limit ? value.substring(0, limit) + '...' : value;
  }
}

<!-- Utilisation -->
<p>{{ description | truncate:100 }}</p>
Pipe async Le pipe async est très puissant : il souscrit automatiquement à un Observable ou une Promise, et se désabonne quand le composant est détruit — évitant les memory leaks.

7 Lifecycle Hooks

Angular appelle ces méthodes à des moments précis du cycle de vie d'un composant.

Ordre d'exécution

export class MonComposant implements OnInit, OnDestroy, AfterViewInit, OnChanges {

  // 1. Appelé avant ngOnInit, à chaque changement d'@Input()
  ngOnChanges(changes: SimpleChanges) {
    console.log(changes); // { nomPropriete: { currentValue, previousValue, firstChange } }
  }

  // 2. Appelé une fois après la création du composant
  ngOnInit() {
    // Initialiser les données, appels API, subscriptions
  }

  // 3. Appelé à chaque cycle de détection de changements
  ngDoCheck() { /* peu utilisé, peut être coûteux */ }

  // 4. Après que le contenu projeté (ng-content) est initialisé
  ngAfterContentInit() {}
  ngAfterContentChecked() {}

  // 5. Après que la vue et les vues enfants sont initialisées
  ngAfterViewInit() {
    // Accès aux @ViewChild disponibles ici
  }
  ngAfterViewChecked() {}

  // 6. Appelé juste avant la destruction du composant
  ngOnDestroy() {
    // Se désabonner des Observables, vider les timers
  }
}
HookQuandUsage typique
ngOnInitUne fois après la créationChargement des données
ngOnChangesÀ chaque changement d'@InputRéagir aux changements de props
ngAfterViewInitAprès init de la vueAccès @ViewChild, DOM, librairies JS
ngOnDestroyAvant destructionUnsubscribe, nettoyage

Nettoyage propre avec DestroyRef Angular 16+

import { inject, DestroyRef } from '@angular/core';
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';

export class MonComposant {
  private destroyRef = inject(DestroyRef);

  ngOnInit() {
    monObservable$.pipe(
      takeUntilDestroyed(this.destroyRef) // unsubscribe auto à la destruction
    ).subscribe(data => { /* ... */ });
  }
}

8 Services & Injection de dépendances

Les services contiennent la logique métier et l'accès aux données. L'injection de dépendances (DI) d'Angular les instancie et les injecte automatiquement là où on en a besoin.

Créer un service

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root' // singleton disponible partout dans l'app
})
export class AuthService {
  private _isLoggedIn = false;

  login(email: string, password: string): boolean {
    // logique de login
    this._isLoggedIn = true;
    return true;
  }

  logout() { this._isLoggedIn = false; }

  get isLoggedIn() { return this._isLoggedIn; }
}

Injecter un service

// Méthode classique : via le constructeur
export class MonComposant {
  constructor(private authService: AuthService) {}
}

// Méthode moderne (Angular 14+) : inject()
export class MonComposant {
  private authService = inject(AuthService);
}

Portée des services — providedIn

ValeurPortéeInstances
'root'Toute l'application1 singleton
'platform'Plusieurs apps sur la même page1 par plateforme
'any'Chaque module lazy-loaded1 par module
Module spécifiqueDans ce module uniquement1 par module
@Component providersCe composant et ses enfants1 par composant

Service avec état (pattern courant)

@Injectable({ providedIn: 'root' })
export class PanierService {
  private items = signal<Produit[]>([]);

  readonly items$ = this.items.asReadonly();
  readonly total = computed(() =>
    this.items().reduce((acc, p) => acc + p.prix, 0)
  );

  ajouter(produit: Produit) {
    this.items.update(items => [...items, produit]);
  }

  vider() { this.items.set([]); }
}

Guards

// auth.guard.ts
import { inject } from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';

export const authGuard: CanActivateFn = () => {

10 Formulaires

Angular propose deux approches : Template-driven (simple, piloté par le HTML) et Reactive (piloté par le TypeScript, plus puissant).

Template-driven

Nécessite FormsModule. La logique est dans le template :

<!-- Template -->
<form #monForm="ngForm" (ngSubmit)="onSubmit(monForm)">
  <input
    name="email"
    [(ngModel)]="email"
    required
    email
    #emailField="ngModel"
  />
  <div *ngIf="emailField.invalid && emailField.touched">
    Email invalide
  </div>
  <button [disabled]="monForm.invalid">Envoyer</button>
</form>
// Composant
email = '';

onSubmit(form: NgForm) {
  if (form.valid) console.log(form.value);
}

Reactive Forms (recommandé)

Nécessite ReactiveFormsModule. La structure est définie en TypeScript :

import { FormBuilder, FormGroup, Validators } from '@angular/forms';

export class LoginComponent {
  private fb = inject(FormBuilder);

  form = this.fb.group({
    email: ['', [Validators.required, Validators.email]],
    password: ['', [Validators.required, Validators.minLength(8)]],
    age: [null, [Validators.min(18), Validators.max(120)]],
  });

  get email() { return this.form.get('email')!; }

  onSubmit() {
    if (this.form.valid) {
      console.log(this.form.value);
    }
  }
}
<!-- Template -->
<form [formGroup]="form" (ngSubmit)="onSubmit()">
  <input formControlName="email" />
  <div *ngIf="email.invalid && email.touched">
    <span *ngIf="email.errors?.['required']">Champ requis</span>
    <span *ngIf="email.errors?.['email']">Format invalide</span>
  </div>
  <button [disabled]="form.invalid">Connexion</button>
</form>

États d'un FormControl

PropriétéDescription
valid / invalidPasse les validateurs ou non
touched / untouchedL'utilisateur a focalisé puis quitté le champ
dirty / pristineL'utilisateur a modifié la valeur
errorsObjet des erreurs actives ({ required: true, email: true... })

FormArray — liste dynamique de contrôles

import { FormArray, FormControl } from '@angular/forms';

form = this.fb.group({
  tags: this.fb.array([this.fb.control('angular')]),

  this.tags.push(this.fb.control(''));

11 HTTP Client

Angular fournit HttpClient pour communiquer avec des APIs REST. Toutes les méthodes retournent des Observables.

Configuration

// app.module.ts
import { HttpClientModule } from '@angular/common/http';

@NgModule({
  imports: [HttpClientModule]
})

Utilisation dans un service

import { HttpClient, HttpHeaders, HttpParams } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({ providedIn: 'root' })
export class UserService {
  private api = 'https://api.example.com';

  constructor(private http: HttpClient) {}

  getAll(): Observable<User[]> {
    return this.http.get<User[]>(`${this.api}/users`);
  }

  getById(id: number): Observable<User> {
    return this.http.get<User>(`${this.api}/users/${id}`);
  }

  create(user: Partial<User>): Observable<User> {
    return this.http.post<User>(`${this.api}/users`, user);
  }

  update(id: number, data: Partial<User>): Observable<User> {
    return this.http.put<User>(`${this.api}/users/${id}`, data);
  }

  delete(id: number): Observable<void> {
    return this.http.delete<void>(`${this.api}/users/${id}`);
  }

  // Avec paramètres et headers
  search(query: string): Observable<User[]> {
    const params = new HttpParams().set('q', query);
    const headers = new HttpHeaders({ 'Authorization': 'Bearer token' });
    return this.http.get<User[]>(`${this.api}/users`, { params, headers });
  }
}

Intercepteurs

Les intercepteurs interceptent toutes les requêtes/réponses HTTP pour ajouter des headers, gérer les erreurs, etc. :

import { HttpInterceptorFn } from '@angular/common/http';

export const authInterceptor: HttpInterceptorFn = (req, next) => {
  const token = localStorage.getItem('token');

  const authReq = req.clone({
    headers: req.headers.set('Authorization', `Bearer ${token}`)
  });

  return next(authReq);
};

// Enregistrement dans app.module.ts
providers: [
  provideHttpClient(withInterceptors([authInterceptor]))
]

12 RxJS & Observables

RxJS est la bibliothèque de programmation réactive utilisée par Angular. Un Observable est un flux de données asynchrone auquel on peut s'abonner.

Observable vs Promise

ObservablePromise
ValeursPlusieurs dans le tempsUne seule
DéclenchementLazy (au subscribe)Eager (immédiat)
AnnulableOui (unsubscribe)Non
Opérateursmap, filter, switchMap....then() uniquement

Opérateurs essentiels

import { map, filter, switchMap, catchError, tap, debounceTime, distinctUntilChanged } from 'rxjs/operators';
import { of, from, combineLatest } from 'rxjs';

// map — transformer chaque valeur
users$.pipe(
  map(users => users.filter(u => u.actif))
)

// switchMap — remplacer par un autre Observable (annule le précédent)
// Parfait pour les recherches en temps réel
searchInput$.pipe(
  debounceTime(300),           // attendre 300ms après la dernière frappe
  distinctUntilChanged(),       // ne pas relancer si la valeur n'a pas changé
  switchMap(query => api.search(query)) // annule la requête précédente
)

// catchError — gérer les erreurs
api.getUsers().pipe(
  catchError(err => {
    console.error(err);
    return of([]); // retourner une valeur par défaut
  })
)

// tap — effet de bord sans modifier la valeur (debug)
api.getUsers().pipe(
  tap(users => console.log('Reçu :', users))
)

// combineLatest — combiner plusieurs observables
combineLatest([user$, roles$]).pipe(
  map(([user, roles]) => ({ ...user, roles }))
)

Subject et BehaviorSubject

import { Subject, BehaviorSubject } from 'rxjs';

// Subject : pas de valeur initiale, n'émet qu'aux abonnés actifs
private action$ = new Subject<string>();
this.action$.next('cliqué');

// BehaviorSubject : a une valeur initiale, les nouveaux abonnés reçoivent la dernière valeur
private user$ = new BehaviorSubject<User | null>(null);
this.user$.next(newUser);
this.user$.getValue(); // valeur synchrone actuelle

// Exposer en lecture seule
readonly user = this.user$.asObservable();

13 Signals Angular 16+

Les Signals sont une alternative plus simple aux Observables pour gérer l'état réactif. Un signal est une valeur qui notifie automatiquement ses dépendants quand elle change.

Concepts de base

import { signal, computed, effect } from '@angular/core';

// signal() — état modifiable
const compteur = signal(0);

compteur();           // lire la valeur → 0
compteur.set(5);      // remplacer
compteur.update(v => v + 1); // modifier à partir de la valeur actuelle

// computed() — valeur dérivée (recalculée automatiquement)
const double = computed(() => compteur() * 2);
double(); // 12 (si compteur = 6)

// effect() — effet de bord déclenché à chaque changement
effect(() => {
  console.log('Compteur changed:', compteur());
});

WritableSignal dans un composant

export class ProduitsComponent {
  produits = signal<Produit[]>([]);
  filtre = signal('');

  produitsFiltres = computed(() =>
    this.produits().filter(p =>
      p.nom.includes(this.filtre())
    )
  );

  charger() {
    this.produitService.getAll().subscribe(data => {
      this.produits.set(data);
    });
  }
}

14 Décorateurs utiles

@Input() et @Output()

Permettent la communication parent ↔ enfant :

// Composant enfant
export class CarteComponent {
  @Input() titre!: string;          // reçoit une valeur du parent
  @Input({ required: true }) id!: number; // obligatoire (Angular 16+)

  @Output() supprime = new EventEmitter<number>();

  onSupprimer() {
    this.supprime.emit(this.id); // envoie l'événement au parent
  }
}

<!-- Parent -->
<app-carte
  [titre]="'Mon titre'"
  [id]="item.id"
  (supprime)="onSupprimer($event)"
></app-carte>

@ViewChild() et @ViewChildren()

Accèdent à un élément ou composant enfant de la vue :

import { ViewChild, ViewChildren, QueryList, ElementRef } from '@angular/core';

export class MonComposant implements AfterViewInit {
  // Accès à un élément du template via la variable de référence #monInput
  @ViewChild('monInput') inputRef!: ElementRef;

  // Accès à un composant enfant
  @ViewChild(CarteComponent) carte!: CarteComponent;

  // Accès à tous les éléments d'un type
  @ViewChildren(CarteComponent) cartes!: QueryList<CarteComponent>;

  ngAfterViewInit() {
    // Disponible seulement ici (pas dans ngOnInit !)
    this.inputRef.nativeElement.focus();
    this.cartes.forEach(c => console.log(c));
  }
}

<!-- Template -->
<input #monInput type="text" />

@HostListener()

Écoute des événements sur l'élément hôte du composant/directive :

export class MonComposant {
  @HostListener('click', ['$event'])
  onClick(event: MouseEvent) {
    console.log('Cliqué !', event);
  }

  @HostListener('window:resize', ['$event'])
  onResize(event: Event) {
    console.log('Fenêtre redimensionnée');
  }

  @HostListener('document:keydown.escape')
  onEscape() {
    this.fermerModal();
  }
}

15 Change Detection

Angular surveille les changements d'état pour mettre à jour la vue. Il existe deux stratégies.

Default (par défaut)

Angular vérifie tout l'arbre de composants à chaque événement (clic, timer, requête HTTP...). Simple mais potentiellement lent sur les grosses apps.

OnPush — optimisation

Angular ne vérifie le composant que si :

  • Un @Input() reçoit une nouvelle référence (pas juste une mutation)
  • Un événement dans ce composant est déclenché
  • Un Observable lié avec async pipe émet une valeur
  • On appelle manuellement markForCheck()
@Component({
  changeDetection: ChangeDetectionStrategy.OnPush
})
export class ListeComponent {
  @Input() items: Item[] = [];

  // ⚠️ Ceci ne déclenchera PAS la détection (mutation du tableau) :
  // this.items.push(newItem);

  // ✅ Ceci déclenchera la détection (nouvelle référence) :
  // this.items = [...this.items, newItem];
}

ChangeDetectorRef — contrôle manuel

import { ChangeDetectorRef } from '@angular/core';

export class MonComposant {
  constructor(private cdr: ChangeDetectorRef) {}

  chargerDonnees() {
    this.data = [...];
    this.cdr.markForCheck();   // marque pour vérification lors du prochain cycle
    // ou
    this.cdr.detectChanges();  // force une vérification immédiate
  }
}

16 Accès au DOM

Angular encourage à éviter la manipulation directe du DOM (pour garder la compatibilité SSR, les tests, etc.). Voici les approches recommandées.

@ViewChild + ElementRef

La façon Angular d'accéder à un élément du template :

export class MonComposant implements AfterViewInit {
  @ViewChild('maDiv') maDiv!: ElementRef<HTMLDivElement>;

  ngAfterViewInit() {
    // Lecture
    console.log(this.maDiv.nativeElement.offsetHeight);

    // Modification (préférer Renderer2 pour la sécurité)
    this.maDiv.nativeElement.style.color = 'red';
  }
}

<!-- Template -->
<div #maDiv>Contenu</div>

Renderer2 — manipulation sécurisée

Renderer2 est l'API Angular recommandée pour modifier le DOM. Elle fonctionne avec le rendu côté serveur (SSR) et est protégée contre les injections XSS :

import { Renderer2, ElementRef } from '@angular/core';

export class MonComposant {
  constructor(
    private renderer: Renderer2,
    private el: ElementRef
  ) {}

  styler() {
    const div = this.el.nativeElement.querySelector('div');

    // Styles
    this.renderer.setStyle(div, 'color', 'blue');
    this.renderer.removeStyle(div, 'color');

    // Classes
    this.renderer.addClass(div, 'actif');
    this.renderer.removeClass(div, 'actif');

    // Attributs
    this.renderer.setAttribute(div, 'aria-label', 'description');

    // Créer / ajouter des éléments
    const span = this.renderer.createElement('span');
    const texte = this.renderer.createText('Bonjour');
    this.renderer.appendChild(span, texte);
    this.renderer.appendChild(div, span);

    // Écouter un événement
    this.renderer.listen(div, 'click', () => console.log('cliqué'));
  }
}

document natif

Quand les APIs Angular ne suffisent pas, on peut injecter DOCUMENT plutôt que d'utiliser le global document directement (meilleure compatibilité SSR) :

import { DOCUMENT } from '@angular/common';
import { inject } from '@angular/core';

export class MonService {
  private document = inject(DOCUMENT);

  scrollToTop() {
    this.document.body.scrollTop = 0;
    this.document.documentElement.scrollTop = 0;
  }

  getElement(id: string) {
    return this.document.getElementById(id);
  }
}
Attention — nativeElement direct Accéder à el.nativeElement directement pour modifier le DOM (innerHTML, style...) expose à des risques XSS et casse le rendu SSR. Préférer Renderer2 sauf pour de la lecture.

17 Animations

Angular propose un système d'animations déclaratif via @angular/animations.

Configuration

// app.module.ts
import { BrowserAnimationsModule } from '@angular/platform-browser/animations';

@NgModule({
  imports: [BrowserAnimationsModule]
})

Concepts clés

import {
  trigger, state, style, animate, transition, keyframes
} from '@angular/animations';

@Component({
  animations: [
    trigger('ouverture', [          // nom de l'animation

      state('ouvert', style({       // état nommé avec ses styles
        height: '*',
        opacity: 1,
      })),
      state('ferme', style({
        height: '0px',
        opacity: 0,
      })),

      // Transition entre états
      transition('ferme => ouvert', animate('300ms ease-in')),
      transition('ouvert => ferme', animate('200ms ease-out')),
      // ou les deux sens en même temps :
      transition('ouvert <=> ferme', animate('250ms')),
    ])
  ]
})
export class AccordeonComponent {
  etat = 'ferme';
  toggle() { this.etat = this.etat === 'ferme' ? 'ouvert' : 'ferme'; }
}

<!-- Template -->
<div [@ouverture]="etat">Contenu</div>
<button (click)="toggle()">Toggle</button>

Animations :enter / :leave

Animez les éléments qui apparaissent/disparaissent avec *ngIf :

trigger('fadeInOut', [
  transition(':enter', [       // apparition (équivaut à void => *)
    style({ opacity: 0, transform: 'translateY(-10px)' }),
    animate('300ms ease-out', style({ opacity: 1, transform: 'translateY(0)' }))
  ]),
  transition(':leave', [       // disparition (équivaut à * => void)
    animate('200ms ease-in', style({ opacity: 0, transform: 'translateY(-10px)' }))
  ]),
])

<!-- Template -->
<div *ngIf="visible" [@fadeInOut]>Apparition animée</div>

Keyframes — animation étape par étape

trigger('rebond', [
  transition('* => actif', [
    animate('600ms', keyframes([
      style({ transform: 'scale(1)',   offset: 0    }),
      style({ transform: 'scale(1.3)', offset: 0.4  }),
      style({ transform: 'scale(0.9)', offset: 0.7  }),
      style({ transform: 'scale(1)',   offset: 1    }),
    ]))
  ])
])

AnimationBuilder — animations programmatiques

Pour animer des éléments sans passer par le template :

import { AnimationBuilder, AnimationPlayer, animate, style } from '@angular/animations';

export class MonComposant {
  private player!: AnimationPlayer;

  constructor(
    private builder: AnimationBuilder,
    private el: ElementRef
  ) {}

  animer() {
    const factory = this.builder.build([
      style({ opacity: 0 }),
      animate('500ms ease-in', style({ opacity: 1 }))
    ]);

    this.player = factory.create(this.el.nativeElement);
    this.player.play();

    this.player.onDone(() => console.log('Animation terminée'));
  }
}
Conseil Pour des animations simples (fade, slide), les transitions CSS pures via [style] ou [class] sont souvent suffisantes et plus performantes. Utiliser @angular/animations pour des séquences complexes ou les animations :enter/:leave.
